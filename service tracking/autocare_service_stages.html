Stage 1
PENDING
Appointment booked
Customer has booked. Vehicle has not yet arrived at the shop. No work has started.
Set by: system on bookingVisible: customer + staff
›
Stage 2
UNDER_INSPECTION
Vehicle inspected
Vehicle has arrived. Staff is assessing the job, identifying parts needed, and building the cost estimate.
Set by: staffTriggers: none yet
›
Stage 3
WAITING_FOR_APPROVAL
Awaiting customer OK
Initial estimate sent. Work is paused until customer approves or declines the cost breakdown.
Set by: staff (on estimate submit)Triggers: push notification
›
Stage 4
IN_PROGRESS
Work underway
Customer has approved. Mechanics are actively working on the vehicle. Staff can add progress notes.
Set by: system on approvalTriggers: push notification
›
Stage 5
COMPLETED
Service done
All work finished. Final invoice generated and sent to customer. Vehicle ready for pickup.
Set by: staffTriggers: final invoice + push
CANCELLED — exit from any stage. Initiated by customer (from PENDING only) or admin/staff (any stage). Triggers cancellation notification. Soft-deletes the appointment record.
Valid transitions
Allowed forward transitions
PENDING → UNDER_INSPECTION
UNDER_INSPECTION → WAITING_FOR_APPROVAL
WAITING_FOR_APPROVAL → IN_PROGRESS
IN_PROGRESS → COMPLETED
Allowed exits
Any stage → CANCELLED (admin/staff)
PENDING → CANCELLED (customer)

Rejected: any backward or skip transition
